# Observation: Write Org output atomically and stop following symlinks

- **Source commit**: `a6a3f549ceb85a78443956007b88a0f3e1197521`
- **Observed at**: `2026-06-28T22:23:44.476Z`
- **Severity**: High
- **Category**: Bug
- **File**: `gemini_to_org.py:2705`
- **Confidence**: 85

## Problem

The Org output is written with `open(output_file, 'w', encoding='utf-8')`
(lines 2704-2706) with no existence check, no atomic temp-file+rename, and no
`O_NOFOLLOW`. When `--output-file` is not given, the path is an auto-generated,
deterministic `YYYYMMDDHHMM-title.org` name (`generate_output_filename`, around
lines 2533-2552).

## Impact

1. **Clobbering**: if the generated name collides with an existing hand-edited
   `.org` file, that file is overwritten with no warning.
2. **Corruption on crash**: a failure mid-`write` leaves a truncated/partial
   file in place of the original.
3. **Symlink following**: if the deterministic target path is a pre-existing
   symlink, the write follows it and overwrites the link target — a (low-
   likelihood) redirection vector in a shared or predictable output directory.

Batch conversion over a directory (`convert_all_gemini_notes.sh`) makes
name collisions across re-runs plausible, raising the practical risk of (1)/(2).

## Suggested Fix

Write to a temp file in the same directory and `os.replace()` into place so the
publish is atomic; open with `O_NOFOLLOW` (or `realpath`-check that the target
stays inside the intended directory). Optionally refuse to overwrite an existing
target unless `--force` or an explicit `--output-file` is supplied.

```python
import os, tempfile
d = os.path.dirname(output_file) or '.'
fd, tmp = tempfile.mkstemp(dir=d, suffix='.org.tmp')
with os.fdopen(fd, 'w', encoding='utf-8') as f:
    f.write(org_content)
os.replace(tmp, output_file)   # atomic on the same filesystem
```

## Verification

Add tests: (a) converting twice does not silently clobber a pre-existing,
differently-sourced `.org` at the same path (or is gated by `--force`); (b) a
symlink planted at the target path is not followed. Simulate a write failure and
confirm the prior file is preserved intact.
