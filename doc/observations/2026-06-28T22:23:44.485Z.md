# Observation: Stop dropping transcript lines that begin with `##`, `---`, or `Dec `

- **Source commit**: `a6a3f549ceb85a78443956007b88a0f3e1197521`
- **Observed at**: `2026-06-28T22:23:44.485Z`
- **Severity**: Medium
- **Category**: Bug
- **File**: `gemini_to_org.py:2489`
- **Confidence**: 65

## Problem

After timestamp/speaker handling, the transcript parser (lines 2489-2493) skips
any remaining line that begins with `##`, `---`, or `Dec ` as "metadata." These
prefixes are not exclusive to export artifacts: real spoken or continuation
content can begin with them — a dictated `---` aside, a wrapped continuation line
that happens to start with `##`, or a sentence beginning "Dec ".

## Impact

Silent loss of transcript words. The Python reviewer verified that both
`"--- some aside ---"` and `"## not-a-timestamp continuation"` are dropped while
an ordinary continuation line is kept. As with the other parsing data-loss paths,
the user gets no indication that content was discarded. (Confidence is moderate
because how often real transcripts contain such lines is unknown — but the
behavior itself is confirmed.)

## Suggested Fix

Restrict the skip to lines that are *actual* Markdown headings (i.e.
`self._parse_markdown_heading(line)` returns truthy) and to the specific known
Google-export artifacts, rather than any line carrying those prefixes. Append
non-heading lines to `current_content` when a timestamp section is open.

## Verification

Add a test placing continuation lines that start with `##` and `---` inside a
timestamped section and assert they appear verbatim in the converted output.
