# Observation: Run the converter once per batch instead of once per file

- **Source commit**: `a6a3f549ceb85a78443956007b88a0f3e1197521`
- **Observed at**: `2026-06-28T22:23:44.479Z`
- **Severity**: High
- **Category**: Performance
- **File**: `convert_all_gemini_notes.sh:50`
- **Confidence**: 90

## Problem

The wrapper loop (lines 50-74) invokes `gemini_to_org.py` once per file. Each
invocation re-runs the `nix-shell` shebang, starts a fresh CPython process,
re-imports the `anthropic` SDK, re-reads the participant DB / vocabulary / prompt
files, and reconstructs the `Anthropic` client(s) in `main()` (lines 2640-2667).
Because transcript task inference is **on by default** here
(`convert_all_gemini_notes.sh:28`), a live `TranscriptTaskInferer` HTTP client is
built for every file. Each process also calls `participant_db.save()`
(line 2709), which `json.dump`s the entire (monotonically growing) DB with
`indent=2, sort_keys=True` even when no participant was added.

## Impact

For N files this is N interpreter cold-starts + N `nix-shell` evaluations + N SDK
imports + N client constructions + N full DB rewrites. The fixed per-process
overhead (easily ~0.5-2s each for nix-shell + `anthropic` import) can dwarf the
actual conversion work, turning a sub-minute directory job into many minutes.
DB I/O grows as O(N·P) (P = cumulative participant count) instead of O(P).

## Suggested Fix

Add a batch mode to `gemini_to_org.py`: accept multiple input paths
(`nargs='+'`), construct the participant DB / `Vocabulary` / `TodoShortener` /
`TranscriptCleaner` / `TranscriptTaskInferer` once, loop over files internally,
and call `participant_db.save()` exactly once at the end (or only when the DB
changed — a dirty flag in `get_id`). Then have the wrapper call the converter a
single time over the glob:

```bash
"$CONVERTER" "${converter_args[@]}" "$DIR"/*"Notes by Gemini.md"
```

(Note: the per-file *model calls* are inherent to the inference feature and are
not the target here — the avoidable cost is process/import/client startup and the
repeated DB serialization.)

## Verification

Time the wrapper over a directory of N ≥ 20 notes before and after the change;
confirm a single Python process is spawned and the DB is written once. Add a
test exercising multi-path invocation.
